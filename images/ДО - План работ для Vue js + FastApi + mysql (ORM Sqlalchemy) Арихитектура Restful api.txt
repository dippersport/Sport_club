Vue js + FastApi + mysql (ORM Sqlalchemy) Арихитектура RestfullApi
Что требуется сделать, поэтапно, на первом месяце работы:

ПЕРВЫЙ МЕСЯЦ РАБОТЫ
Этап-1. Развертывание админки, задаем основы фронтенда будущего проекта:
1. Разворачиваем на ВАШЕМ хостинге новый проект (на нем работаем до завершения работ по Этапу-1).
2. Пишем фронтенд с нуля. Верстка - повторяем всё для публичных страниц и страниц Личного кабинета (нам нужен такой же внешний вид как у сайта-примера), в том же стиле.
3. Локализуем проект (категории, наполнение - всё берем с сайта-примера https://online24.ru). 
4. Настраиваем формы авторизации, капчу, прелоадер, микроразметку, прорабатываем оплату (Робокасса, Ю-Мани), систему скидок (всё как в сайте-примере, нового не 
изобретаем). Оповещение по E-Mail.
5. Ловим и устраняем баги, оптимизируем структуру БД. Перепроверяем и настраиваем СЕО проекта.
6. Тем временем дизайнер доделывает макеты фронтенда (уже под конкретную тематику - Недвижимость) в Фигме.

Этап-2. Стилизация фронтенда под основную тематику Заказчика (Недвижимость):
1. Из макетов дизайнера (в Фигме) - оптимизируем и обрезаем картинки, иконки. Переносим в верстку нашего проекта.
2. Вносим в верстке по отдельным страницам мелкие правки (изменений в версии дизайнера не много - чуть меняем форму-размеры-цвет некоторых полей-блоков и их 
расположение, картинки, иконки, подписи). 
3. Забиваем в БД тестовые товары - для тестирования функционала проекта (это я сам сделаю).
4. Страница модераторов (в админке) - разбираемся какие нам нужны для нормальной работы Доски объявлений, настраиваем совместно (Вы и я)
5. То же самое - по страница админа проекта в админке.
6. Проверяем страницы проекта по скорости загрузки (ГуглСпидТест), оптимизируем. 

Этап-3. Проработка Сателлитов:
1. Используем уже готовый движок с Этапа-2.
2. Пишем и настраиваем API обмена данными, правим структуру БД (чтобы 2 таких проекта могли работать в паре: Основной с кабинетом модератора - и управляемый им
 Сателлит-тематический)
3. Делаем еще 1 копию нашего готового шаблона-Сателлита. Стилизуем (под свои тематики - Недвижимость, Страна, Туризм). Исходники дизайна в Фигме будут готовы.
4. Публикуем все Сателлиты на Вашем сервере. Настраиваем их на совместную работу, ловим и исправляем баги.
5. Выбираем оптимальную конфигурацию необходимой нам Виртуальной машины на хостинге заказчика для проекта в целом (Основной проект + Сателлиты + сервисный функционал).
6. Выгружаем релиз на хостинг с ВМ. Тестируем хостинг, настраиваем домены и оповещения. Высылаем Заказчику готовый архив, проводим оплату работ Этапов-1-2-3.

Этап-4. Доработка проекта:
1. Настраиваем выполнение задач из ЛК по расписанию (например, через Cron) - публикация нового поста, удаление нами раннее опубликованного поста. И для авторов постов, и для модератора.
2. Добавляем расширенную Статистику в проект (скрипты и пара графиков в ЛК пользователей по показам постов).
3. Дорабатываем Авто-модерацию (проверку на соблюдение правил) новых пользователей, постов-товаров, магазинов.
4. Настраиваем кабинет модератора (что выводить для контроля и управления - обсуждаем в рабочем порядке. Смотрим результат, вносим правки). 1 модератор обслуживает
 несколько Сателлитов.
5. Аккаунту пользователя добавляем управление несколькими магазинами со своими парами логин-пароль для входа (основа в движке уже для этого есть, но сейчас 
управление только одним магазином)
6. Выгружаем релиз на сервер Заказчика. Высылаем Заказчику полный архив, проводим оплату работ Этапа-4.


ПЕРСПЕКТИВЫ ДАЛЬНЕЙШЕГО СОТРУДНИЧЕСТВА
На второй месяц - у нас будет дальнейшее развитие этого же проекта. Работа с соцсетями, ботами простыми и средними, планировщик публикаций и т.п.. Второй месяц 
уже будет оплачиваться исходя из бюджета 100.000 руб
Третий месяц - еще глубже в соцсети залезем - всякие инвайты, более сложные боты и т.п. Третий месяц и далее - оплачивается уже из бюджета 150.000+ руб.
И так далее.
